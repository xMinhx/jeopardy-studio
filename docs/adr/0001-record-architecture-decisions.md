# ADR 1: Record Architecture Decisions

## Status

Accepted

## Context

We need a way to record important architectural decisions made during the development of the Jeopardy Scoreboard project. This helps current and future contributors understand why certain choices were made.

## Decision

We will use Architecture Decision Records (ADRs) to document significant architectural choices. These records will be stored in the `docs/adr/` directory.

## Consequences

- Architectural decisions are transparent and documented.
- New contributors can ramp up faster by reading past decisions.
- There is a historical record of the project's evolution.
